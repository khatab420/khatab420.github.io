2. Google AdMob – Advertising Data Collection (Updated)

To keep WhatsApp Tools free for all users, we integrate Google AdMob SDK to display advertisements. AdMob is a trusted advertising platform by Google. The AdMob SDK may collect certain device and interaction data to provide relevant, non-personalized or personalized ads (based on user consent region). Below we detail the exact data collected, purpose, and user controls.

2.1 What AdMob Collects (based on Google's AdMob policies)

  • Advertising ID (AAID): Google Play Advertising ID – resettable by the user. Used for ad frequency capping, analytics, and (if consented) personalized ads.
  • IP Address: Approximate location (city/region level) to serve geo-relevant ads, not precise GPS.
  • Device Information: Device make/model, operating system version, screen size, language settings, and mobile network carrier.
  • App interaction data: Ad impressions, ad clicks, and limited engagement events (to measure performance).
  • Android identifiers (non-resettable not accessed): AdMob does NOT collect IMEI, MAC address, or Android ID for advertising purposes under modern Google Play policies.

2.2 Ad Personalization & User Consent (GDPR / CCPA / Google’s requirements)

Depending on your location (EEA, UK, Brazil, etc.), we respect your choice regarding personalized ads:

  • Personalized ads: Use your Advertising ID and interests to show relevant ads. You can disable this anytime via device settings.
  • Non-personalized ads: Based on contextual information (app content, approximate location) without using interest profiles.
  • Children-directed treatment: If we detect the user is under the age of consent, AdMob will serve only non-personalized ads with no behavioral targeting.

For users in the European Economic Area (EEA), UK, or Switzerland, we display a consent dialog (Google UMP SDK) before loading ads, asking for permission to use cookies/AAID for personalized advertising. You can change your choice anytime in the app's privacy settings.

🔐 Opt-out of Ad Personalization (global)
Go to Settings → Google → Ads → Opt out of Ads Personalization on your Android device. This resets your Advertising ID and restricts interest-based ads across all apps.

2.3 AdMob SDK & Third-Party Data Sharing

The AdMob SDK may share limited data with Google's ad mediation partners only when required for ad serving. WhatsApp Tools does not add any additional tracking or analytics SDK (such as Firebase Analytics, Crashlytics, or Facebook SDK). All data handling by AdMob is governed exclusively by Google's Privacy Policy and Google AdMob Data Disclosure.

Important: WhatsApp Tools does not share, sell, or transfer any local user content (statuses, messages, reply rules) to AdMob or any third party. The advertising ID is used solely by AdMob, not by us.

3. Feature-Specific Data Handling (100% Local)

3.1 Status Saver Feature

  • Reads WhatsApp status folders only after you grant storage permission.
  • Saves statuses to Downloads/WhatsAppTools/Statuses/ – never uploaded anywhere.
  • No media leaves your device. No analytics about which status you save.

3.2 Auto Reply Feature

  • Notification Listener permission (explicit user approval) detects incoming WhatsApp notifications.
  • All rule matching happens locally using Room database. No text messages stored in cloud.
  • We never log or transmit the content of your messages or contacts.

3.3 Message Scheduler & Direct Chat

  • Scheduled messages stored offline in local database; removed upon uninstall.
  • Direct chat uses standard wa.me deep links; no phone numbers saved.

4. Permissions Explained (For Transparency)

  • READ_EXTERNAL_STORAGE / READ_MEDIA_IMAGES/VIDEO: Access WhatsApp status folder.
  • WRITE_EXTERNAL_STORAGE: Save statuses to Downloads folder.
  • POST_NOTIFICATIONS (Android 13+): Show local notifications (status saved, auto-reply triggers).
  • Notification Listener Service: Enables Auto Reply; optional and disabled by default.
  • INTERNET: Required only for loading AdMob ads and opening wa.me links. No personal data transmitted.
  • RECEIVE_BOOT_COMPLETED: Restore scheduled messages after reboot.
  • SCHEDULE_EXACT_ALARM / USE_EXACT_ALARM: Trigger scheduled messages at precise time (user-defined).

5. Data Storage and Security – Local-First Approach

All user-generated content (auto-reply rules, scheduled messages, settings) is stored in app-private directories (Room database & DataStore). No cloud backup. Saved statuses are in public Downloads folder under your control. Uninstalling WhatsApp Tools deletes every piece of app-generated data except statuses saved manually (you can delete them via file manager).

AdMob-related data (AAID, IP address) is managed by Google’s secure infrastructure, subject to Google’s data retention policies. We have no server-side logs.
